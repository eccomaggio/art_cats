One-off orders

Instructions

  1. This template is for placing one-off orders only.
  2. Enter your order column by column, row by row. Mandatory columns are highlighted in bold. Data validation rules apply to blue columns - choose an option from the drop-down menu to proceed.
  3. If convenient, all bibliographic information for the order can be pasted into the 'Title' column cell to avoid unnecessary clicking.
  4. Columns without row outlines mean that the last value given in a column will apply to all subsequent rows. Information in these columns does not need to be duplicated for each row. The reverse is true of outlined rows.
  5. 'Additional info' column can be used to provide any additional information about the order, from preferred vendor to the preferred edition of a title.
  6. Please share your order as an attachment via email. A quick way to do this is to click 'File', then 'Share'. This will prompt you to open the app version of Outlook, save a copy of the spreadsheet and add it as an attachment giving the file name as the subject line.

N.B. The options in the Fund code and Location depend on the information provided for the Subject consultant and Library respectively. If you cannot find the budget or location you are looking for, please check first that you have chosen the correct Subject Consultant for that budget, or the correct Library for the location. If any drop down menus need to be updated, please contact acqservices@bodleian.ox.ac.uk.

Glossary

'Fund code'
The ALMA fund code assigned to each budget. Fund code mapping from Aleph is available here. (Opens in default browser.) Note that you must select the subject consultant first to see the list of the relevant fund codes.
'Hold for'
Place a hold request on the item for this person at the earliest opportunity. Separate to 'Notify'. This should always be the 7 digit university number (printed on the staff card). We can't accept emails for two main reasons: 1) personal emails are problematic in the light of UK data protection legislation; 2) users may have multiple official emails, not all of which are linked to their ALMA account.
'Item policy'
An explanation of ALMA item policies is here. (Opens in default browser.)
'Library' and 'location'
An explanation of the changes from Aleph are available here. (Opens in default browser.) Note that you must select the Library first to gain access to its location codes.
Bibliographic information
Ideally, creator (/ author), publishing date & ISBN would be in their separate boxes, but it is fine to just paste them into the bibliographic information box.
Additional instructions
This is very wide-ranging: include such topics as number of copies required, the preferred vendor if you have one, preferred format (paperback / hardback), etc. Basically, give us any extra information that you think will help us track down exactly the item you want.
'Notify'
Notify this person of an item's availability at the earliest opportunity. This should always be the 7 digit university number (printed on the staff card). We can't accept emails for two main reasons: 1) personal emails are problematic in the light of UK data protection legislation; 2) users may have multiple official emails, not all of which are linked to their ALMA account.
'Reporting code'
Here. (Opens in default browser.)
