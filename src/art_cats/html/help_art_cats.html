Spreadsheet cataloguing

This app is designed for non-cataloguers to enter information which can then be transformed into bibliographic records (Marc 21 RDA records) for upload to ALMA. It deals with most of the tricky markup and linking aspects but there are a few things that you need to know in order to use it effectively. The following information should help you:

First, check title against SOLO!!!

This is essential to avoid creating duplicate records. If a record already exists, do not enter the details on the spreadsheet.

NB. It may be more efficient to check a number of titles against SOLO and then switch to enter the details into the spreadsheet, rather than jumping between SOLO and the app each time.

Capitalisation

Data is generally entered in sentence case (even if it is all in capital letters on the resource) unless there is a reason for it to be capitalised, e.g. an acronym, proper noun, or entering text in say German, where all nouns must be capitalised.

Incomplete records

Records must include the following minimum information to be saved:

  • the language(s)
  • a title
  • a country and place of publication
  • a publisher
  • the height and number of pages
  • a year of publication
  • the barcode

These mandatory fields are marked with an * in their title on the form.

Records must also be complete to be locked.

More help

There is detailed help below about each of the input fields. You can either scroll down or click on a title in the form: that will automatically bring the help topic into view.

Locking / Editing

By default, complete records are locked for safety. Toggle the unlock button if you want to edit them further.

Red Backgrounds

Don't worry if the background of the field you are typing in suddenly turns pink: it just means that you are overwriting an existing record.

If this is your aim, continue to type and press Save when you've amended the record and it will be saved in its new form.

If you were overtyping a record by mistake, just go backwards or forwards to the next record and your overtyping will be forgotten.

Alternatively, press New if you want to keep your changes as a new record.

Backups

Everytime a record is submitted, it is backed up to a file unimaginatively named [original file name].new.csv

Deleting records

If you Clear a record and then Save it, it will be deleted.

Item policy

See HOL Notes

Sublibrary

Enter the sublibrary which the materials belong to (BODBL, RSLBL, ARTBL or similar). Team Leaders can advise.

Language of resource

This is required in order to sort the records at a later stage in the process. Enter the language in which the resource is written (e.g. English or French). If the item has multiple languages, this should be entered as, e.g. English/German. (You can also use spaces or commas as separators.) Put them in order of priority, for instance: if the resource is mainly in Chinese, but has a parallel title in English, it should be labelled Chinese/English. This allows the app to process titles etc. correctly for the particular language.

You can use LLC language codes if you wish, but if in doubt, use the full English form.

ISBN(s)

Enter if applicable.

Title

Transcribe the title using sentence case. Include any initial article, e.g. The, A, La, Der (this will be automatically tagged as non-filing).

If you want to add an alternative version--typically to expand an acronym or to represent a number that has been spelt out in letters as digits (e.g. CNG -> Classical Numismatics Group or forty-two -> 42), type the full alternative title after the official one, separating the two with the sign *//*

If a title divides naturally (because of punctuation) into several parts, put the first part into the main Title field and ALL the others into the Subtitle field. The subtitle doesn't need to be further subdivided. For example: "Osian's dare to care: an initiative for the adolescent girl -- a charity fundraising gala event" should be split: Title="Osian's dare to care", Subtitle="an initiative for the adolescent girl -- a charity fundraising gala event". Note also that the punctuation (here a colon) separating the title from the subtitle should be omitted.

Subtitle

Transcribe the subtitle, omitting any initial article. Begin with a lower-case letter (unless the first word of the subtitle requires a capital letter e.g. a proper noun in English or a German noun). See "Title" for alternate subtitles.

Parallel title

(For dual language items only)

Transcribe the parallel subtitle, omitting any initial article. Begin with a lower-case letter unless the first word of the parallel subtitle requires a capital letter (as for Subtitle). See "Title" for alternate parallel subtitles.

Transliteration

Only for all languages with non-Latin scripts. Make sure you follow the relevant Library of Congress transcription guidelines for each language.

Country of publication

Enter the country of publication.

NB. If the resource was published in a place which is now in a different country, enter the present-day name of that country (e.g. Bratislava is now in Slovakia although at time of publication it was in Czechoslovakia)

The spelling, capitalisation etc. don't really matter, so USA, U.S.A., usa, the United States of America will all work. You will get an error if the name isn't recognised. Also, if you know the LCC country code, you can use this (capitalisation is ignored).

For places in the US, Canada, UK and Australia, a 'state' is expected. For the UK, 'state' means 'England / Ireland / Scotland / Northern Ireland'. For some common cities, such as New York and London, the state can be left out.

State

This is needed only for states that have a 3-letter country code: The USA, the UK, Canada & Australia. Transcribe from the resource. NB. In the case of the UK, this means simply specifying Northern Ireland, England, Scotland or Wales.

Place of publication

Transcribe from the resource.

Publisher name

Transcribe from the resource. If working in a non-European script, use the publisher's English name. In the rare event of there being two or more, equal publishers, simply add the one listed first and ignore the other(s).

Date of publication

Enter the year of publication ONLY from the resource.

Do not record day/month.

Add a question mark for conjectured dates [i.e. 2015?]

Copyright date

If the resource has a copyright date, input the date (without the Â© symbol: this will be inserted automatically).

Pagination

Enter the number of pages, e.g. vii, 504. Do not add the words pages , pp., etc.

If the number of pages is not given explicitly, add a question mark after your guesstimation / count, i.e. 175?.

Do not record illustrations, maps etc.

Size

Measure height of book and round up to nearest centimetre but do not include cm, cms, centimeters, etc., as this will be added automatically] i.e, 22 cm

Illustrations

Select None if there are no illustrations and Partial if there are illustrations, even quite extensive, but the text could in principle stand alone without them.Choose Full if the text would lose its meaning without the illustrations, as is the case with many auction catalogues.

Series title

Transcribe from the resource.

Series enumeration

Transcribe from the resource.

Note

Add notes for anything unusual or relevant about the book, e.g. Title from cover or listings on the title page in the case of art catalogues, if it helps to differentiate resources with very similar titles.

Sale code

(Additional field for Auction Catalogues)

This is a numeric or alphanumeric code assigned to a sale, usually found on the catalogue's title page but sometimes further in. Be wary of numbers found on the spine, which are sometimes incomplete.

If there are multiple sale codes, these should be entered as, e.g. ORCHID27, LILY35

Where they are already included in the title, that is enough. Think of the Sale code as being something someone might search for to find the item on the catalogue so e.g. 38A230503 might be helpful (although it is only the auction number and date) but just 27A isn't that useful. The Sotheby catalogues show how distinctive some of them are.

Date of sale

(Additional field for Auction Catalogues)

Use the format YYYYMMDD for a single date. If the date is incomplete, record as much of the date as is known and use hyphens for the missing digits.

If there are multiple dates, these should be entered as, e.g. 20210724, 20210731

HOL notes

Add notes pertaining to the copy in your hand, e.g.a donor note; Missing pages 4-5.

If the book's item policy should be "Fixed_Week_Loan", then type ***f week** (exactly as is, one space) as the FIRST thing in this field. Otherwise, the default is "Use_in_Library_Only".

Donor note

For example: Donated by the Ashmolean Museum or Anonymous donor.

Barcode

Attach the barcode onto the item in the top left-hand corner of the front cover and scan the barcode into the field.

If you type *dummy* into this field, it will allow you to save an incomplete record.
